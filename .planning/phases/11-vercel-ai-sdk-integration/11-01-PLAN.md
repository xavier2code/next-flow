## 1. backend/app/schemas/chat.py -- Request schema

```python
"""Schema for the SSE chat endpoint."""

from pydantic import BaseModel, Field


class ChatRequest(BaseModel):
    """Request body for the SSE chat endpoint.

    Compatible with Vercel AI SDK useChat POST body.
    The `id` field is the conversation ID (path param takes precedence).
    The `messages` field is sent by useChat but the backend uses LangGraph checkpoint
    for actual conversation state -- this field is for protocol compliance only.
    """
    id: str | None = None
    messages: list[dict] | None = None  # UIMessage[] from useChat -- ignored by backend
    message: str | None = None  # Alias for content (useChat may send either)
    content: str = Field(min_length=1, max_length=10000, description="User message text")
```

## 2. backend/app/api/v1/chat.py -- SSE endpoint

Create a new module with a POST endpoint that returns an SSE stream in Data Stream Protocol v2 format.

**Endpoint**: `POST /api/v1/conversations/{conversation_id}/chat`

**Auth**: Use `get_current_user` dependency (Bearer token, same as messages.py).

**Request body**: `ChatRequest` with `content` field.

**Response**: `StreamingResponse` with:
- Content-Type: `text/event-stream`
- Headers: `x-vercel-ai-ui-message-stream: v1`, `Cache-Control: no-cache`, `X-Accel-Buffering: no`

**Flow**:
1. Validate conversation ownership (same pattern as messages.py: `ConversationService.get_for_user`)
2. Save user message to DB (same pattern as messages.py: `ConversationService.add_message`)
3. Resolve agent config from conversation's linked agent (same pattern as messages.py)
4. Yield Data Stream v2 events by streaming LangGraph astream_events:
   - `start` event: `{"type":"start","messageId":"msg_{uuid}"}`
   - `start-step` event: `{"type":"start-step"}`
   - Stream events from graph.astream_events (reuse existing logic from event_mapper.py)
   - Map LangGraph events to Data Stream v2 protocol parts (see mapping below)
   - `finish` event: `{"type":"finish","finishReason":"stop"}`
   - `data: [DONE]\n\n` terminator
5. After stream completes, save assistant message to DB (same pattern as messages.py `_trigger_agent_execution`)

**LangGraph to Data Stream v2 mapping** (implement in the SSE generator):

| LangGraph Event | Protocol Part |
|-----------------|---------------|
| (stream start) | `{"type":"start","messageId":"msg_{uuid}"}` |
| (step start) | `{"type":"start-step"}` |
| `on_chat_model_stream` with `chunk.tool_calls` | `{"type":"tool-input-start","toolCallId":"{tc.id}","toolName":"{tc.name}"}` |
| `on_chat_model_stream` with `chunk.content` (NOT in think tag) | `{"type":"text-start","id":"text_{uuid}"}` on first text, then `{"type":"text-delta","id":"text_{uuid}","delta":"{content}"}` |
| `on_chat_model_stream` with `chunk.content` (INSIDE think tag) | `{"type":"reasoning-start","id":"reason_{uuid}"}` on first reasoning, then `{"type":"reasoning-delta","id":"reason_{uuid}","delta":"{content}"}` |
| `on_tool_start` | `{"type":"tool-input-available","toolCallId":"{event.name}","toolName":"{event.name}","input":{event.data.input}}` |
| `on_tool_end` | `{"type":"tool-output-available","toolCallId":"{event.name}","output":{event.data.output}}` |
| `on_chain_end` (no parent_ids) | `{"type":"finish","finishReason":"stop"}` |
| (error) | `{"type":"error","error":"{error message}"}` |

**ThinkTagFilter reuse**: Import `ThinkTagFilter` from `app.api.ws.event_mapper`. The existing `process()` method returns `[{"type": "thinking", "data": {"content": ...}}, {"type": "chunk", "data": {"content": ...}}]`. Map `"type": "thinking"` to `reasoning-delta` events and `"type": "chunk"` to `text-delta` events. Do NOT modify event_mapper.py in this plan.

**SSE format**: Each event is `data: {JSON}\n\n`. Use `json.dumps(event, ensure_ascii=False)`. After the finish event, send `data: [DONE]\n\n`.

**Abort support**: The SSE generator should check if the client has disconnected. Use `request.is_disconnected()` or catch `asyncio.CancelledError` to gracefully stop the LangGraph stream.

**Message persistence**: After the stream generator completes (after `finish` event is yielded), save the collected assistant text to the database using `async_session_factory()` and `ConversationService.add_message()`. This is the same pattern as the existing `_trigger_agent_execution` in messages.py. Wrap in try/except so persistence failures don't break the stream.
