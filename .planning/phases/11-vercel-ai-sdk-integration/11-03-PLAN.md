<objective>
Remove the WebSocket infrastructure (endpoint, ConnectionManager, Redis pub/sub listener) and the fire-and-forget async message pattern from the backend, now that SSE chat endpoint handles streaming directly.

Purpose: The WebSocket + Redis pub/sub bridge was needed for the old architecture where the POST endpoint returned 202 and streamed via WebSocket. With the new SSE endpoint, the response is streamed directly in the same HTTP request. This cleanup eliminates ~200 lines of infrastructure code and removes the Redis pub/sub dependency for chat.

Output: Clean backend with SSE-only chat, no WebSocket or pub/sub infrastructure.
</objective>

The simplified `send_message`:

```python
@router.post("/conversations/{conversation_id}/messages", status_code=202)
async def send_message(
    conversation_id: str,
    data: MessageCreate,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
) -> Response:
    conversation = await ConversationService.get_for_user(
        db, str(current_user.id), conversation_id
    )
    if conversation is None:
        raise NotFoundException(message="Conversation not found")

    await ConversationService.add_message(
        db, conversation_id, role="user", content=data.content
    )
    await db.commit()

    return Response(status_code=202)
```
