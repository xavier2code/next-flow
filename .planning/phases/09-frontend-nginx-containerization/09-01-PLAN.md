  <action>
    Create `frontend/Dockerfile` with the following exact content (per D-03 and FRNT-01):

    ```dockerfile
    # ---- Stage 1: Build React SPA ----
    FROM node:22-alpine AS builder

    WORKDIR /app

    # Copy dependency specs first for Docker layer caching
    COPY package.json package-lock.json ./
    RUN npm ci

    # Copy source and build
    COPY . .
    RUN npm run build

    # ---- Stage 2: Serve with Nginx ----
    FROM nginx:1.27-alpine

    # Remove default Nginx static assets
    RUN rm -rf /usr/share/nginx/html/*

    # Copy built SPA from builder
    COPY --from=builder /app/dist /usr/share/nginx/html

    # Copy custom Nginx server config
    COPY nginx/default.conf /etc/nginx/conf.d/default.conf

    EXPOSE 80

    # Nginx runs as 'nginx' user by default (set in /etc/nginx/nginx.conf)
    CMD ["nginx", "-g", "daemon off;"]
    ```

    Key decisions implemented:
    - `node:22-alpine` per CLAUDE.md tech stack (Node 22 LTS) -- D-03
    - `nginx:1.27-alpine` per RESEARCH.md verified version (1.27.5) -- D-03
    - `npm ci` (not `npm install`) for deterministic builds from lockfile
    - COPY package.json + package-lock.json first for layer caching (dependency changes don't invalidate source layer)
    - Non-root user: official nginx:alpine already runs as `user nginx` -- no USER directive needed
    - No HEALTHCHECK in Dockerfile (Nginx static serving is trivial; Phase 10 adds compose-level health checks)
    - References `nginx/default.conf` which will be created in Plan 09-02

    After creating the file, verify the build succeeds:
    ```bash
    docker build -t nextflow-frontend frontend/
    ```
    The build will fail at the `COPY nginx/default.conf` line because that file does not exist yet (Plan 09-02 creates it). This is expected. To verify the build stages work, temporarily comment out the COPY nginx line, build, and then restore it. Alternatively, create a placeholder empty file at `frontend/nginx/default.conf` to unblock the build.
