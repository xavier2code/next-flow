## Color

Theme: Neutral base (achromatic) with oklch color space. Supports light and dark mode via `.dark` class.

| Role | Value (Light) | Value (Dark) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `oklch(1 0 0)` -- white | `oklch(0.145 0 0)` -- near-black | Background, surfaces |
| Secondary (30%) | `oklch(0.97 0 0)` -- light gray | `oklch(0.205 0 0)` -- dark gray | Cards, sidebar, popover |
| Accent (10%) | `oklch(0.97 0 0)` -- light gray | `oklch(0.269 0 0)` -- medium gray | Hover states, selected items, active sidebar items |
| Destructive | `oklch(0.577 0.245 27.325)` -- red | `oklch(0.704 0.191 22.216)` -- red | Delete actions, error states, destructive buttons |

Accent reserved for: sidebar-active item background, button hover/focus states, selected list items, input focus rings. Not used for primary buttons (primary is black/white).

Source: `frontend/src/index.css` CSS custom properties (verified 2026-04-01)

## Phase 9 Nginx Visual Impact Notes

While Nginx configuration is infrastructure, it has one visual correctness concern:

| Concern | Resolution |
|---------|------------|
| SPA client-side routing must work through Nginx | `try_files $uri $uri/ /index.html` in default.conf (FRNT-02) |
| SSE streaming must arrive token-by-token | `proxy_buffering off` + `proxy_cache off` in /api/v1/ location (D-01) |
| Static assets must be compressed | gzip enabled for text/css, application/javascript, application/json, image/svg+xml (FRNT-06) |

These are correctness requirements, not design decisions. The visual appearance of the SPA is identical whether served by Vite dev server or Nginx.
