    Create `frontend/nginx/default.conf` with the following exact content (per D-01, D-02, FRNT-02, FRNT-03, FRNT-04 superseded, FRNT-06):

    ```nginx
    server {
        listen 80;
        server_name _;

        root /usr/share/nginx/html;
        index index.html;

        # Gzip compression (FRNT-06)
        gzip on;
        gzip_vary on;
        gzip_proxied any;
        gzip_comp_level 6;
        gzip_min_length 256;
        gzip_types
            text/plain
            text/css
            text/javascript
            application/javascript
            application/json
            application/xml
            application/rss+xml
            image/svg+xml;

        # API reverse proxy (FRNT-03) with SSE passthrough (D-01, FRNT-04 superseded)
        location /api/v1/ {
            proxy_pass http://backend:8000;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            # SSE: disable buffering so events stream immediately
            # Backend sends X-Accel-Buffering: no, but Nginx must also be configured
            proxy_buffering off;
            proxy_cache off;

            # Keep connections alive for long agent workflows
            proxy_read_timeout 300s;
        }

        # SPA fallback routing (FRNT-02)
        location / {
            try_files $uri $uri/ /index.html;
        }
    }
    ```

    Key decisions implemented:
    - **FRNT-02**: `try_files $uri $uri/ /index.html` serves index.html for all non-file routes (SPA client-side routing)
    - **FRNT-03**: `proxy_pass http://backend:8000` routes /api/v1/ to the backend container (service name `backend` defined in Phase 10 docker-compose)
    - **FRNT-04 SUPERSEDED by D-01**: No WebSocket Upgrade/Connection headers. SSE endpoint (POST /api/v1/conversations/{id}/chat) is under /api/v1/ prefix, so `proxy_buffering off` + `proxy_cache off` in the same location block handles SSE streaming
    - **FRNT-06**: gzip enabled for text/css, application/javascript, application/json, image/svg+xml and related MIME types
    - **D-02**: Minimal config scope -- no security headers (Phase 10), no cache-control headers (Phase 10), no JSON structured logging (Phase 10)
    - `gzip_proxied any` ensures gzip works for proxied API responses
    - `gzip_min_length 256` avoids compressing tiny responses (overhead > savings)
    - `gzip_comp_level 6` balances CPU vs compression ratio (level 6 is the sweet spot)
    - `proxy_read_timeout 300s` (5 min) for long agent workflows -- SSE is request-scoped so this is per-request, not persistent like WebSocket
    - `proxy_http_version 1.1` for keepalive to backend
    - `server_name _` matches any hostname (appropriate for Docker Compose internal networking)
    - This is a `conf.d/default.conf` replacement -- the default `nginx.conf` includes `conf.d/*.conf`, preserving worker_processes, error_log, and MIME type includes

    IMPORTANT: Do NOT include `text/event-stream` in `gzip_types`. SSE responses must not be compressed (the event stream format is already efficient). The current list deliberately excludes it.

    IMPORTANT: Do NOT include WebSocket-specific headers (Upgrade, Connection). Phase 11 replaced WebSocket with SSE per D-01.
