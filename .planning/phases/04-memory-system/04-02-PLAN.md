   - Import `json`, `time`, `structlog`, `redis.asyncio as aioredis`, `asyncio`
   - Import `AIMessage`, `HumanMessage`, `SystemMessage` from `langchain_core.messages`
   - Import `get_llm` from `app.services.agent_engine.llm`
   - Define class `ShortTermMemory`:
     - `__init__(self, redis: aioredis.Redis, window_size: int = 20, summary_threshold: int = 15, ttl: int = 86400)`:
       Per D-01: window_size = last N messages. Per D-04: ttl = 86400 (24h sliding TTL). Per Claude's discretion: summary_threshold = 15 (when raw messages exceed this, compress older ones).
       Store as `self._redis`, `self._window_size`, `self._summary_threshold`, `self._ttl`.
     - `async def add_message(self, thread_id: str, role: str, content: str) -> None`:
       a. Build key: `f"nextflow:memory:short:{thread_id}"` (per D-03)
       b. Create message dict: `{"role": role, "content": content, "timestamp": time.time()}`
       c. Serialize: `json.dumps(msg)`
       d. Use Redis pipeline: ZADD key with score=timestamp, ZREMRANGEBYRANK to trim to window_size, EXPIRE key with self._ttl (per D-04)
       e. After pipeline execute, check if compression needed: if ZCARD > self._summary_threshold, fire `asyncio.create_task(self._compress(key))` (per D-05)
     - `async def get_context(self, thread_id: str) -> dict`:
       a. Build key: `f"nextflow:memory:short:{thread_id}"`
       b. Use pipeline: GET summary key (`f"{key}:summary"`), ZRANGE messages key 0 -1
       c. Parse results: summary = results[0] or None, messages = [json.loads(m) for m in results[1]]
       d. Return `{"summary": summary, "messages": messages}`
     - `async def _compress(self, key: str) -> None`:
       a. Get all messages from the sorted set (ZRANGE 0 -1)
       b. If count <= summary_threshold, return (nothing to compress)
       c. Split into "old" (first N - summary_threshold + 5) and "recent" (remaining)
       d. Call LLM to generate summary of old messages: use get_llm() with a system prompt like "Summarize the following conversation messages into a concise summary retaining key facts, decisions, and context. Output plain text only."
       e. Store summary via SET `{key}:summary` with same TTL as main key
       f. Remove old messages from sorted set via ZREMRANGEBYRANK 0 (old_count - 1)
       g. Wrap entire method in try/except, log errors but never raise (fire-and-forget per D-05)

     - `async def should_store(self, new_fact: str, existing_facts: list[str]) -> bool`:
       Per D-15: LLM-based deduplication.
