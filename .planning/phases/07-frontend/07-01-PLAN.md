<interfaces>
<!-- Backend API surface the frontend must consume -->

Auth endpoints (bare response, no envelope):
- POST /api/v1/auth/register {email, password, display_name?} -> UserResponse (201)
- POST /api/v1/auth/login {email, password} -> TokenResponse {access_token, refresh_token, token_type}
- POST /api/v1/auth/refresh {refresh_token} -> TokenResponse
- POST /api/v1/auth/logout {refresh_token?} -> {message: string} (requires auth)
- GET /api/v1/auth/me -> UserResponse (requires auth)

Envelope format:
- EnvelopeResponse: {data: T, meta: null}
- PaginatedResponse: {data: T[], meta: {cursor: string|null, has_more: bool}}

Error format: {error: {code: string, message: string}}

Entity schemas:
- UserResponse: {id: UUID, email, display_name?, avatar_url?, role, created_at}
- ConversationResponse: {id: UUID, title, is_archived, created_at, updated_at}
- AgentResponse: {id: UUID, name, system_prompt?, llm_config?, created_at, updated_at}
- MessageResponse: {id: UUID, conversation_id, role, content, created_at}
- SkillResponse: {id: UUID, name, description?, version, skill_type, status, permissions?, manifest?, created_at, updated_at}
- MCPServerResponse: {id: UUID, name, url, transport_type, status, config?, created_at, updated_at}
- SkillToolResponse: {name, namespaced_name, description?, input_schema?}
- MCPToolResponse: {name, namespaced_name, description?, input_schema?}

WebSocket: ws://host/ws/chat?token={jwt}
- Server-push only, 5 event types as {type, data}:
  - thinking: {content}
  - tool_call: {name, args, id?}
  - tool_result: {name, result}
  - chunk: {content}
  - done: {thread_id} or {error}

Settings:
- SystemConfigResponse: {available_providers: string[], default_provider, default_model}
</interfaces>

    **6. Create API client (`frontend/src/lib/api-client.ts`):**
    Build a fetch wrapper with these behaviors:
    - Base URL: '' (Vite proxy handles routing)
    - Method helpers: `get<T>(path)`, `post<T>(path, body)`, `patch<T>(path, body)`, `del(path)`
    - Auth header injection: read `access_token` from auth store getter
    - 401 handling: on 401 response, attempt token refresh via `POST /api/v1/auth/refresh`, retry original request once. If refresh fails, clear auth state and redirect to `/login`.
    - Refresh queue: while refresh is in-flight, queue concurrent 401 requests and retry after refresh completes (prevent race condition).
    - Response parsing: unwrap `data` field from EnvelopeResponse. Auth endpoints return bare objects (no unwrap).
    - Error parsing: extract `{error: {code, message}}` and throw typed error.
    - Proactive refresh: export a `startTokenRefreshTimer()` function that schedules `setTimeout` at 12 minutes (80% of 15-min token lifetime) to refresh tokens. Call this on successful login.

      - `tool_call` -> append to toolCallEntries, set sidePanelOpen=true
      - `tool_result` -> append to toolResultEntries
      - `chunk` -> append content to streamingMessage.content
      - `done` -> finalize streamingMessage into messages array, set isStreaming=false

    **9. Create WebSocket hook (`frontend/src/hooks/use-websocket.ts`):**
    - Accept `token: string | null` param
    - When token is provided, connect to `ws://${window.location.host}/ws/chat?token=${token}` (use window.location.host so Vite proxy works)
    - Track connectionStatus: 'connecting' | 'connected' | 'disconnected'

    - On reconnect: refresh token first, then reconnect with new token
    - Return `{ connectionStatus }`
    - Handle cleanup on unmount (close socket, clear timers)
