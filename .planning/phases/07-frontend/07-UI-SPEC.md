## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, input icon padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, card header padding |
| xl | 32px | Layout gaps, major section separation |
| 2xl | 48px | Page-level top padding, major section breaks |
| 3xl | 64px | Page-level spacing, empty state illustration offset |

Exceptions:
- Activity bar icon touch targets: 48px (CONTEXT D-02 -- icon column must be comfortably clickable)
- Input box height: 64px (CONTEXT D-08 -- fixed bottom input needs comfortable multi-line space)
- Side panel width: 320px (non-standard, but needed for tool call readability)

### TailwindCSS Mapping

Use Tailwind's default spacing scale which already aligns to 4px multiples:
`p-1` (4px), `p-2` (8px), `p-4` (16px), `p-6` (24px), `p-8` (32px), `p-12` (48px), `p-16` (64px).

Custom values via `@theme` for the exceptions:
- `--spacing-activity-bar: 48px`
- `--spacing-input-box: 64px`
- `--spacing-side-panel: 320px`
- `--spacing-sidebar: 260px`

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm leading-relaxed font-normal` |
| Label | 12px | 500 (medium) | 1.5 | `text-xs font-medium` |
| Heading | 16px | 600 (semibold) | 1.2 | `text-base font-semibold` |
| Display | 20px | 600 (semibold) | 1.2 | `text-lg font-semibold` |

### Usage Rules

- **Body (14px/400)**: Message content, list items, descriptions, table cells, form helper text
- **Label (12px/500)**: Form labels, badge text, tab labels, sidebar section headers, metadata (timestamps, IDs)
- **Heading (16px/600)**: Card titles, panel headers, section titles, conversation titles in sidebar
- **Display (20px/600)**: Page titles, welcome message heading, auth page titles

### Markdown Response Typography

Agent response text (rendered via react-markdown) inherits body styling with these additions:
- Code blocks: `text-xs` (12px), monospace font (`font-mono`), background `muted`, padding `p-4`
- Headings within markdown: scale as `text-base` (h3), `text-lg` (h2) -- do not exceed display size
- Links: accent color, underline on hover

## Color

Dark theme is default (CONTEXT D-20). All colors below are dark-mode values. Light mode is a 1:1 inversion via shadcn CSS variable swap.

### Semantic Color Map

| Role | CSS Variable (shadcn) | Dark Hex | Usage |
|------|----------------------|----------|-------|
| Dominant (60%) | `--background` | `hsl(240 10% 3.9%)` | Page background, main area |
| Secondary (30%) | `--card` / `--popover` | `hsl(240 10% 3.9%)` with `--border` at `hsl(240 3.7% 15.9%)` | Sidebar, cards, panels, activity bar |
| Accent (10%) | `--primary` | `hsl(0 0% 98%)` | CTA buttons, active tab indicator, selected item highlight, send button, focus rings |
| Muted | `--muted` | `hsl(240 3.7% 15.9%)` | Empty state background, code blocks, disabled states |
| Destructive | `--destructive` | `hsl(0 62.8% 30.6%)` | Delete confirmations, error messages, remove actions |

### Accent Reserved For (explicit list)

Accent color (`--primary` white on dark) is reserved for:
1. Primary CTA buttons (Send Message, Save Agent, Upload Skill, Register Server)
2. Active tab indicator (underline on current tab)
3. Selected conversation highlight in sidebar
4. Focus ring on form inputs
5. Agent response bubble background (subtle -- `--primary` at 5% opacity)

NOT used for: links in markdown (use a distinct blue), badges, icons in activity bar, decorative elements.

### Semantic Status Colors

| Status | Color | Hex (dark) | Usage |
|--------|-------|------------|-------|
| Success / Connected | Green | `hsl(142 76% 36%)` | MCP server connected badge, skill enabled badge |
| Warning / Connecting | Yellow | `hsl(48 96% 53%)` | MCP server connecting badge, skill loading state |
| Error / Disconnected | Red (destructive) | `hsl(0 62.8% 30.6%)` | MCP server disconnected badge, skill error, connection lost |
| Info / Neutral | Muted foreground | `hsl(240 5% 64.9%)` | Tool call status, thinking indicator, timestamps |

### Theme Toggle

- Dark mode: default, no class needed
- Light mode: toggle adds `class="light"` to `<html>`, shadcn CSS variables swap automatically
- Toggle location: Settings page (CONTEXT D-04, D-20)
- Persist preference in Zustand `persist` middleware (localStorage for theme only, not tokens)

## Interaction Patterns

### Authentication Flow

1. User lands on `/login` (full-screen card)
2. Enter email + password -> click "登录"
3. On success: tokens stored in memory (CONTEXT D-19), redirect to `/`
4. On error: inline error message below form
5. Link to `/register` for new users
6. Token refresh: proactive refresh at 80% of token lifetime (12min for 15min token)
7. On refresh failure: redirect to `/login`

### Chat Flow

1. WebSocket connection established on app load (after auth) at `ws://host/ws/chat?token={jwt}`
2. User selects or creates conversation in sidebar
3. Message input: Shift+Enter for newline, Enter to send
4. On send: `POST /api/v1/conversations/{id}/messages` (returns 202), message appears immediately in stream
5. WebSocket events arrive: `chunk` events append to streaming text, `thinking`/`tool_call`/`tool_result` appear in side panel
6. `done` event: finalize message, stop streaming indicator
7. Connection status dot in header shows WebSocket state (green/yellow/red)

### Management Flow

1. Click Management icon in activity bar
2. Tab bar shows Agent / Skills / MCP
3. Left panel: scrollable list of items
4. Click item -> right panel shows details
5. Inline editing: fields editable directly in detail panel, "Save" button to persist
6. New item: "Create" button opens empty detail panel on right

### Agent Configuration Fields

| Field | Component | Validation |
|-------|-----------|------------|
| Name | Input (text) | Required, 1-100 chars |
| Model | Select (dropdown) | Required, options from backend settings API |
| System Prompt | Textarea | Optional, max 4000 chars |
| Temperature | Slider (0.0-2.0, step 0.1) | Default 0.7 |

### Skill Upload Flow

1. Click "上传技能" button
2. File dialog opens, filtered to `.zip` files only
3. Select ZIP -> `POST /api/v1/skills` with `multipart/form-data`
4. Loading spinner during upload
5. On success: skill appears in list with "disabled" status
6. User clicks enable toggle -> skill activates

### MCP Server Registration

| Field | Component | Validation |
|-------|-----------|------------|
| Name | Input (text) | Required, 1-100 chars |
| URL | Input (url) | Required, valid URL |
| Transport | Select (streamable_http / sse) | Default: streamable_http |

After registration, connection status auto-updates via polling (every 30s refresh from server status API).
