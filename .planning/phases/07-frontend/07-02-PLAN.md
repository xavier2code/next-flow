    handleSend function (must handle both cases: existing conversation and no conversation):
    ```typescript
    const handleSend = async (text: string) => {
      // Step 1: Ensure we have a conversation
      let conversationId = useChatStore.getState().currentConversationId;

      if (!conversationId) {
        // No current conversation (welcome screen) -- create one first
        const createMutation = useCreateConversation();
        const result = await createMutation.mutateAsync({ title: text.slice(0, 50) });
        // Result is EnvelopeResponse<Conversation> already unwrapped by apiClient
        conversationId = result.data.id;
        useChatStore.getState().setCurrentConversation(conversationId);
        // Navigate to the new conversation URL
        navigate(`/conversations/${conversationId}`);
      }

      // Step 2: Optimistically show the user message
      useChatStore.getState().addUserMessage(text);

      // Step 3: Send message to backend (returns 202)
      await apiClient.post(`/api/v1/conversations/${conversationId}/messages`, { content: text });

      // Step 4: Mark streaming as started
      // (WebSocket events will populate the streaming response)
    };
    ```

    Key details for the no-conversation branch:
    - `useCreateConversation().mutateAsync(...)` returns the mutation result (EnvelopeResponse already unwrapped by apiClient)
    - Extract `data.id` from the response to get the new conversation UUID
    - Call `setCurrentConversation(conversationId)` so subsequent sends use the existing conversation
    - Navigate to `/conversations/${conversationId}` so the URL reflects the real conversation
