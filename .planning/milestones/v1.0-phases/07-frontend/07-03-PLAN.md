<!-- Backend API surface for management -->

Skills:
- POST /api/v1/skills (multipart/form-data, file field) -> EnvelopeResponse<Skill> (201)
- GET /api/v1/skills -> PaginatedResponse<Skill>
- GET /api/v1/skills/{id} -> EnvelopeResponse<Skill>
- PATCH /api/v1/skills/{id} {description?} -> EnvelopeResponse<Skill>
- DELETE /api/v1/skills/{id} -> 204
- POST /api/v1/skills/{id}/enable -> EnvelopeResponse<Skill>
- POST /api/v1/skills/{id}/disable -> EnvelopeResponse<Skill>
- GET /api/v1/skills/{id}/tools -> EnvelopeResponse<ToolInfo[]>

MCP Servers:
- POST /api/v1/mcp-servers {name, url, transport_type?, config?} -> EnvelopeResponse<MCPServer> (201)
- GET /api/v1/mcp-servers -> PaginatedResponse<MCPServer>
- GET /api/v1/mcp-servers/{id} -> EnvelopeResponse<MCPServer>
- PATCH /api/v1/mcp-servers/{id} {name?, url?, transport_type?, config?} -> EnvelopeResponse<MCPServer>
- DELETE /api/v1/mcp-servers/{id} -> 204
- GET /api/v1/mcp-servers/{id}/tools -> EnvelopeResponse<ToolInfo[]>

Settings:
- GET /api/v1/settings -> EnvelopeResponse<UserSettings>
- PATCH /api/v1/settings {preferences} -> EnvelopeResponse<UserSettings>
- GET /api/v1/settings/system -> EnvelopeResponse<SystemConfig> (no auth)
</interfaces>

    - `useUploadSkill()` -- useMutation: `POST /api/v1/skills` with `FormData` (append file as 'file' field). Content-Type must NOT be set manually (let browser set multipart/form-data boundary). On success invalidate ['skills']
