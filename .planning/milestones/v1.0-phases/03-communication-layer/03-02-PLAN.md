    - `class ConnectionManager`: singleton managing active WebSocket connections per user
    - Internal structure: `self._connections: dict[str, list[WebSocket]] = {}` (user_id -> list of active connections)
    - `connect(user_id: str, websocket: WebSocket) -> None`: add websocket to user's connection list. Create list if user not tracked.
    - `disconnect(user_id: str, websocket: WebSocket) -> None`: remove websocket from user's list. Delete user entry if list becomes empty.

    **3. Create event mapper** in `backend/app/api/ws/event_mapper.py`:
    - Import `AIMessageChunk` from `langchain_core.messages`
    - `async def map_stream_events(graph, user_input: str, thread_id: str, config: dict) -> AsyncGenerator[dict, None]`
    - Accept parameters: `graph: CompiledStateGraph`, `user_input: str`, `thread_id: str`, `config: dict` (contains `configurable: {"thread_id": ...}`)
    - Build input: `{"messages": [HumanMessage(content=user_input)]}` -- import HumanMessage from langchain_core.messages
    - Call `graph.astream_events(input, config, version="v2")`
    - Iterate events, map based on event["event"]:
      - `"on_chat_model_stream"`: get `chunk = event["data"].get("chunk")`. If `isinstance(chunk, AIMessageChunk)`:
        - If `chunk.tool_calls`: for each tc, yield `{"type": "tool_call", "data": {"name": tc.get("name", ""), "args": tc.get("args", {}), "id": tc.get("id", "")}}`
        - Elif `chunk.content`: yield `{"type": "chunk", "data": {"content": chunk.content}}`
      - `"on_tool_start"`: yield `{"type": "tool_call", "data": {"name": event["name"], "args": event["data"].get("input", {})}}`
      - `"on_tool_end"`: yield `{"type": "tool_result", "data": {"name": event["name"], "result": event["data"].get("output")}}`
      - `"on_chain_end"`: if `not event.get("parent_ids")` (graph-level), yield `{"type": "done", "data": {"thread_id": thread_id}}`
      - Custom events: check if `event["event"].startswith("on_custom_event")` or if event name matches a custom event pattern. Yield `{"type": "thinking", "data": {"content": event["data"]}}`. Handle both `on_custom_event` prefix and the `adispatch_custom_event` pattern.
    - Wrap the entire iteration in try/except. On any exception, yield `{"type": "done", "data": {"error": str(exc)}}` to signal error before terminating.
    - Add `import structlog` and `logger = structlog.get_logger()` for logging unmapped events at debug level.
